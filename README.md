# DentApp

DentApp — современный стоматологический онлайн-сервис, позволяющий пациентам записываться на приём, проходить предварительную диагностику и управлять историей посещений. Для врачей предусмотрен интерфейс управления расписанием.

## Основные функции

- **Аутентификация** — регистрация и вход в личный кабинет (в демо-версии данные сохраняются в `localStorage`, в полноценной — в базе данных через API).
- **Запись на приём** — пошаговый выбор услуги, врача, даты и времени.
- **Предварительная диагностика** — опрос о симптомах с формированием рекомендаций.
- **Личный кабинет** — просмотр ближайших приёмов, статистики посещений и быстрые действия.
- **История посещений** — фильтрация и детальный просмотр прошлых визитов.
- **Расписание для врачей** — календарь с доступными и занятыми слотами.

## Технологии

**Фронтенд:**
- HTML5, CSS3 (адаптивная вёрстка)
- JavaScript (vanilla)
- Font Awesome для иконок

**Бэкенд и база данных:**
- PostgreSQL — реляционная база данных (схема описана в `data.sql`)
- REST API (Node.js + Express / Python + FastAPI) по спецификации OpenAPI (Swagger)
- JWT для аутентификации

## Структура файлов

| Файл | Назначение |
|------|------------|
| `DentApp.html` | Основная разметка страницы |
| `style.css` | Все стили |
| `script.js` | Логика фронтенда (навигация, формы, взаимодействие с API/`localStorage`) |
| `data.sql` | SQL-скрипт для создания структуры базы данных |
| `swagger.yaml` | Спецификация OpenAPI (документация и контракт API) |
| `server.js` | Простой HTTP-сервер для локального запуска фронтенда |

## Запуск проекта

### Демо-режим (только фронтенд, данные в `localStorage`)
1. Скачайте или склонируйте репозиторий.
2. Откройте файл `DentApp.html` в браузере (рекомендуется использовать локальный сервер, например Live Server в VS Code).

### Полноценный режим (с базой данных и API)
1. Установите PostgreSQL и выполните скрипт `data.sql` для создания таблиц.
2. Установите зависимости бэкенда (Node.js или Python) и запустите сервер API (порт 3000).
3. Откройте `DentApp.html` через локальный сервер, предварительно настроив в `script.js` базовый URL API (например, `http://localhost:3000/api`).
4. Для просмотра документации API перейдите по адресу `http://localhost:3000/api-docs` или откройте `index.html` (загружает `swagger.yaml` через Swagger UI).

## Использование

- **Гость** может просматривать главную страницу и возможности сервиса.
- **Зарегистрированный пользователь** получает доступ к личному кабинету, записи на приём, диагностике и истории.
- В демо-режиме данные хранятся в `localStorage`, при первом входе можно создать тестовый аккаунт.
- В полноценном режиме данные сохраняются в PostgreSQL, доступны после авторизации через API.

## Особенности

- Все страницы переключаются без перезагрузки (SPA-подход).
- Формы проходят базовую валидацию.
- Интерфейс адаптирован под мобильные устройства.
- Для врачей доступна вкладка «Расписание» (управление слотами через API).
- API полностью документирован в формате OpenAPI (Swagger).
